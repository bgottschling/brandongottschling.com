> **Why**  
> One home for research notes, projects, and writing—lightweight, fast, and easy to grow. It’s a “living workshop” for shipping in public.

## Stack Overview

- **Stack:** Next.js 15 (App Router), MDX + filesystem content, Tailwind, shadcn/ui, `remark-gfm`, `rehype-slug`/`autolink-headings`, Vercel.  
- **Design:** warm, minimal UI; card components; animated `BackgroundNetworkStable` canvas with “harmony” moments.

---

## What changed in v0.2

**Trust & Credibility**
- Added public trust pages: **About**, **Contact**, **Privacy Policy**, **Terms of Use**.
- Footer now surfaces trust links with copyright + “last updated.”  
- Optional `noindex` on Privacy/Terms (kept public, not prioritized in search).

**Navigation**
- **Mobile:** replaced hamburger with a **Command Palette** menu (`shadcn/ui` Command + Dialog).  
  - Instant filtering, grouped links (Main / Trust / External), support for `⌘K / Ctrl+K` and `/`.  
- **Desktop:** kept lean top-nav (Research, Projects, Blog, CV, About); trust links live in footer.

**Performance**
- **Lazy-loaded** the mobile menu via a **client wrapper** to keep `layout.tsx` server-only.  
- Added `tailwindcss-animate`; icons via `lucide-react`.  
- Smaller initial JS, quicker hydration on mobile.

**Structured Data (SEO & LLM-friendly)**
- **Person** JSON-LD in the header (GitHub/LinkedIn/Twitter sameAs).  
- Added **WebSite** JSON-LD; prepared `SearchAction` stub for when site search ships.

**Accessibility**
- “Skip to content” link; `aria-label`s on nav; keyboard shortcuts documented; focus styles intact.

**Dev Experience**
- Standardized UI under `src/components/ui/*` (shadcn).  
- Switched to **pnpm** to avoid peer-dep conflicts and speed up installs.

## Highlights (implementation notes)

- **Footer:** `app/components/Footer.tsx`, rendered in `app/layout.tsx` after `{children}`.  
- **Mobile menu:** `src/components/MobileNav.tsx` + `src/components/MobileNavClient.tsx` (client wrapper).  
- **shadcn/ui:** generated `button`, `dialog`, `command` → `src/components/ui/*`.  
- **Tailwind:** enabled `tailwindcss-animate` in `tailwind.config.ts`.  
- **JSON-LD:** kept `Person`; added `WebSite` block next to it; `SearchAction` commented until `/search` exists.  
- **Privacy/Terms:** MDX supports `export const metadata = { robots: { index: false, follow: false } }` if desired.

## Notes from the build

- **RSC boundaries:** dynamic imports that require `ssr:false` live in a tiny **client wrapper**, not in `layout.tsx`.  
- **Content globs:** Tailwind `content` globs include `src/app`, `src/components`, and MDX content dirs.  
- **Discoverability:** Trust pages are public + linked in footer; **About/Contact** remain indexable.  
- **LLM signals:** clear titles, stable slugs, structured data, internal links from About → Projects/Research.

---

## Takeaways

1. **Trust is a feature.** Clear About/Contact + visible legal pages reduce friction for humans *and* models.  
2. **Command palettes make great mobile navs.** Fast, searchable, and scale with site growth.  
3. **Respect server/client separation.** Keep SSR clean; lazy-load interactive extras.  
4. **Ship structure first.** Footer, schema, and MDX scaffolding unlock steady iteration.  
5. **Build once, reuse everywhere.** Centralized UI (shadcn) + tokens keeps changes consistent.
